租金區間試算表

本專案提供一套依照臺中市最新租金單價表（獨立套房／整層戶）
並依規定套用 屋齡調整、室內裝修加成、區位加成、設備調整金額 的完整線上試算工具。

使用者可輸入「行政區、屋齡、坪數、租金、調整加分」即時計算是否符合規範。

本工具採用 純前端 HTML + JavaScript，可直接部署於 GitHub Pages。

🚀 功能特色
✅ 1. 自動判斷套房／非套房

坪數 ≤ 15 坪 → 套房（使用獨立套房單價表）

坪數 > 15 坪 → 非套房（使用整層戶單價表）

使用者無需選擇類型。

✅ 2. 最新臺中市租金單價表（含屋齡調整）

依你提供的完整兩張表格建立資料模型：

0–4 年

5–9 年

10–14 年

15–19 年

20–24 年

25–29 年

30–34 年

35–39 年

40 年以上

每個行政區皆有 9 格單價，依屋齡自動套用。

✅ 3. 上限規則完整內建

套房：最高不得超過 21,000 元／月

非套房：最高不得超過 35,000 元／月
超過者直接顯示「無法承作」。

✅ 4. 額外加分（另行顯示、不與原本金額混合）

符合規定之三種加值來源：

🟦 區位調整（表二）
級距	調幅
優質區位	+5%
中上區位	+3%
中下區位	0%
🟩 室內裝修（表三）
級距	調幅
中等裝修	+5%
簡易裝修	+3%
未裝修	0%
🟧 設備調整金額（表四）

套房上限：1,000 元／月

非套房上限：2,000 元／月

加分區只有在未達標時顯示（符合時不顯示）。

✅ 5. 計算邏輯

依行政區 × 屋齡 × 類型查出「每坪單價」

最高租金＝每坪單價 × 坪數（取整數、無條件進位）

若最高租金 ≥ 現在租金 → O

否則進入「加分區」做第二次判斷：

最高租金（含加分） = 原最高租金 + 裝修加分 + 區位加分 + 設備加分


結果顯示 O / X（加分後也不一定能過）。

114/11/8 # 租金比較工具 🏠

這是一個將 Excel 規則轉換成網頁的工具。  
使用者只要輸入 **行政區、是否為套房、屋齡、坪數、現在租金**，  
系統會自動計算「最高租金（租金區間 × 坪數）」並判斷結果 **O / X**。

---

## 功能 ✨
- 行政區下拉選單（內建台中各行政區）
- 套房 / 非套房選擇
- 屋齡自動對應分段（0–5、5–10、10–15…40+）
- 自動計算 **最高租金**
- 判斷：
  - 若 **最高租金 ≥ 現在租金** → O（綠色）
  - 否則 → X（紅色）
- 顯示對應的「屋齡分段標籤」（例如 `5–10 套房`）
- 完全前端運行，**不需要伺服器**，支援 GitHub Pages

---

## 使用方式 💻

### 1. 本機使用
1. 下載 `index.html`
2. 在電腦上直接 **雙擊開啟**（用瀏覽器打開）
3. 開始輸入資料並測試 O / X 判斷

---

### 2. GitHub Pages 部署
1. 建立一個新的 GitHub Repository，例如 `rent-checker`
2. 把 `index.html` 上傳到 repo 根目錄
3. 在 repo 頁面點選 **Settings → Pages**
   - **Source** 選「Deploy from a branch」
   - Branch 選 `main`（或 `master`） + `/ (root)`
4. 儲存後，稍等數十秒，GitHub 會產生一個網址  
   （例如 `https://你的帳號.github.io/rent-checker`）
5. 用瀏覽器打開該網址，就能使用工具

---

## 範例輸入 📝
- 行政區：西屯區  
- 是否為套房：yes  
- 屋齡：6  
- 坪數：20  
- 現在租金：25000  

👉 結果：會顯示對應分段 `5–10 套房`，計算最高租金，並判斷 O 或 X。

## 修改說明 🔧
若要修改基礎數據，可在 `index.html` 中找到：
- `HOUSEHOLD` → 各行政區戶數基準  
- `STUDIO` → 各行政區獨立套房基準  
- `ADJUST_RATES` 與 `BRACKETS` → 屋齡調整率與分段設定  

直接編輯這些常數即可。
